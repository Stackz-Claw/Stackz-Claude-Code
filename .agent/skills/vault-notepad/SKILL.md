# Vault Notepad Skill

Bridges the Obsidian vault to the brainstorm workflow. The Idea Notepad 
is where you jot ideas between sessions — this skill reads them in and 
enriches them into proper brainstorm seeds.

---

## NOTEPAD FILE LOCATION

```
~/.stackz/vault/Radar/Idea Notepad.md
```

If this file doesn't exist, create it with the template below on first use.

---

## NOTEPAD TEMPLATE

When creating the notepad for the first time:

```markdown
# Radar — Idea Notepad

This is your scratchpad. Write anything here in Obsidian.
The brainstorm workflow reads this file at session start.

---

## 🧠 Raw Ideas
<!-- Drop rough ideas here. One line or a paragraph. Doesn't matter. -->


---

## 🔗 Links to Explore
<!-- URLs you want researched. One per line. -->


---

## 😤 Frustrations & Gaps
<!-- Things that annoy you. Problems you've hit. Tools that are missing. -->


---

## 💡 Observations
<!-- Things you've noticed in the market, conversations, or your own work. -->


---

## 📌 Seeds for Next Session
<!-- Specific directions you want Radar to explore next time. -->


```

---

## READING THE NOTEPAD

When called by the brainstorm workflow or user:

1. Read `~/.stackz/vault/Radar/Idea Notepad.md`
2. Parse each section separately
3. Extract actionable content:

```
VAULT NOTEPAD CONTENTS
======================
Raw Ideas:      [N items found]
Links:          [N URLs found]
Frustrations:   [N items found]
Observations:   [N items found]
Seeds:          [N items found]

--- RAW IDEAS ---
[list each, numbered]

--- LINKS TO EXPLORE ---
[list each URL with any inline notes]

--- FRUSTRATIONS & GAPS ---
[list each]

--- OBSERVATIONS ---
[list each]

--- SEEDS FOR NEXT SESSION ---
[list each]
```

If the file is empty or all sections are empty:
```
Vault notepad is empty. Open ~/.stackz/vault/Radar/Idea Notepad.md 
in Obsidian to jot ideas between sessions.
```

---

## WRITING TO THE NOTEPAD

When user says "add to my notepad" or "note this in the vault":

1. Read the current notepad
2. Append the new content under the appropriate section
3. Write back to the file
4. Confirm: "✓ Added to [section] in your vault notepad"

Infer the right section:
- URL → Links to Explore
- Problem / complaint / gap → Frustrations & Gaps
- Market observation / trend → Observations
- Specific direction for Radar → Seeds for Next Session
- Rough concept → Raw Ideas

---

## WRITING SESSION OUTPUTS TO VAULT

After a completed brainstorm session, write a summary note:

File: `~/.stackz/vault/Radar/Sessions/brainstorm_[YYYY-MM-DD].md`

```markdown
# Brainstorm Session — [DATE]

## Idea Developed
**Name:** [name]  
**Tagline:** [tagline]  
**Verdict:** [GO / NEEDS_WORK / NO_GO]  
**Leverage Score:** [score]/10  

## Summary
[2-3 sentence summary of the idea]

## Key Signals Found
[bullet list of strongest signals from the sweep]

## Stress Test Risks
[bullet list from the stress test]

## Stackz Take
[final verdict paragraph]

## Sources Used
[list of key URLs / X posts that informed the idea]

## Bookmarks Consumed
[list of bookmark IDs that were marked used this session]

## Seeds Used From Notepad
[list of notepad items that contributed to this idea]

---
*Generated by Radar brainstorm workflow*
```

Create `~/.stackz/vault/Radar/Sessions/` if it doesn't exist.

After writing, clear the **Seeds for Next Session** section in the 
Idea Notepad (those seeds were used). Leave all other sections intact.

---

## OPERATING RULES

1. Never overwrite notepad content — always append to the right section
2. Preserve formatting — the user writes in Obsidian and may have custom structure
3. If the vault directory doesn't exist at `~/.stackz/vault/`, tell the user:
   "Vault not found at ~/.stackz/vault/ — make sure your Obsidian vault is 
   synced here or update the path in this skill."
4. The notepad is personal scratchpad — don't try to parse it too rigidly.
   If a section is missing, treat unclassified content as Raw Ideas.
5. Session output notes are read-only after writing — don't modify past sessions
