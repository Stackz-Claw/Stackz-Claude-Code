description: Run a new idea through the full Stackz idea intake and evaluation pipeline
---

I have a new idea to evaluate. Run the full Stackz idea intake protocol:

1. Give me your honest gut take on this idea RIGHT NOW — before any research. Commit to a view. Is this worth pursuing or not?

2. Package it:
   - idea_id: idea_[today's date]_[sequence]
   - origin: user
   - stackz_interpretation: [what you understand this to be]
   - priority: normal

3. Run Radar's evaluation pipeline:
   STEP 1 — Quick viability (2 min): Does this pass basic kill criteria? Is there demand signal? Can we build it with our stack? Is anyone already doing this well?
   Kill criteria: revenue < $500/mo realistic | domain expertise we don't have | dominant funded incumbent | moral flag | > $1k upfront | one-time revenue only
   Output: promising / questionable / dead

   STEP 2 — Market scan (only if not dead): Find demand evidence, top 3-5 competitors + pricing, market size signal, timing assessment.

   STEP 3 — Stress test (only if market looks good): Find 3 ways this fails. Identify hardest technical challenge. Rough unit economics.

   STEP 4 — Score on Leverage Matrix (0-10):
   - Revenue potential (25%): realistic MRR at 6 months
   - Build complexity (20%): how hard to build MVP
   - Competitive moat (20%): defensibility
   - Market timing (20%): is the window open now
   - Operational fit (15%): fits our current stack and skills
   Apply preference adjustments based on owner's decision history.
   Only package as formal proposal if score ≥ 6.5.

4. Return the full idea report with: radar_verdict, leverage_score, key_findings, stress_test_risks, stackz_take, and options (approve / reject / modify / table).
