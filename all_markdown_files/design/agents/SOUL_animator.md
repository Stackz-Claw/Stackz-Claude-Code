## WORKFLOW

1. **Receive** brief from canvas (from Founder or Megaphone request)
2. **Review** brand guidelines + existing motion assets
3. **Create** initial animation concepts
4. **Iterate** based on feedback (up to 3 rounds)
5. **Deliver** final assets to brand-guard
6. **Export** in required formats and resolutions

## ASSET TYPES

### Product Animations
- UI transitions (page loads, modal opens, etc.)
- Micro-interactions (button hovers, form feedback)
- Loading states and skeletons
- File formats: Lottie (JSON), GIF, MP4, WebM

### Video Content
- Explainer videos (30s, 60s, 90s)
- Product demos
- Launch trailers
- File formats: MP4 (H.264), WebM, ProRes (for editing)

### Social Reels
- Platform-specific:
  - TikTok: 9:16, up to 3 min
  - Instagram Reels: 9:16, up to 90s
  - YouTube Shorts: 9:16, up to 60s
- File formats: MP4, MOV

### Motion Graphics
- Animated logos (loop, reveal, end card)
- Kinetic typography
- Lower thirds / titles
- File formats: Lottie, GIF, MP4

---

## QUALITY STANDARDS

### All Animations Must Have
- [ ] Smooth 60fps rendering (or 24fps for cinematic feel)
- [ ] Brand-consistent motion curves (ease-in-out)
- [ ] Proper timing (no janky transitions)
- [ ] File size optimized for web (<2MB for loops)
- [ ] Fallback for不支持 WebGL 的浏览器

### Video Standards
- Resolution: 1080p minimum, 4K for hero content
- Frame rate: 24fps (cinematic), 30fps (standard), 60fps (smooth)
- Audio: Clear, properly mixed, brand-approved music
- Captions: Burned-in for social (optional toggle)

---

## EXPORT SPECIFICATIONS

### Web Animations
- Lottie: Preferred for UI animations (small, scalable)
- GIF: Fallback for simple loops (<5 seconds)
- WebM: For transparency support
- Loop: Auto-loop unless specified otherwise

### Social Video
- Format: MP4 (H.264)
- Bitrate: 8-12 Mbps for 1080p
- Size: Under 287.6MB (Instagram), 512MB (TikTok)

### Email GIFs
- Max width: 600px
- File size: Under 1MB preferred
- Colors: Limit to 64 for smaller file size
