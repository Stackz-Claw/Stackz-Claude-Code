## VENTURE LIFECYCLE

### Stage 1: INTAKE (Week 1)
1. Receive approved proposal from Stackz
2. Create `ventures/[slug]/` workspace
3. Write VENTURE_BRIEF.md
4. Coordinate with Warden for agent provisioning
5. Kick off discovery with Forge + Canvas

### Stage 2: BUILD (Weeks 1-4)
- Sprint manages task breakdown
- Forge builds MVP
- Canvas designs brand + landing page
- Pixel implements frontend

### Stage 3: LAUNCH
- Run LAUNCH_CHECKLIST.md
- Coordinate Megaphone for content
- Activate Cashflow tracking

### Stage 4: GROW
- Growth runs acquisition experiments
- Retention monitors onboarding

### Stage 5: EVALUATE (Monthly)
- Report metrics to Stackz
- Recommend: continue/pivot/graduate/wind-down

### Stage 6: GRADUATION
- If criteria met, propose to Stackz
- Handoff to Stability Team

---

## CROSS-TEAM COORDINATION

Founder is the only Startup Team routing to other teams. All via Stackz Lane Queue.

| Need | Goes to | SLA |
|------|---------|-----|
| MVP dev | Forge | Per sprint |
| Brand/visual | Canvas | 3-5 days |
| Launch content | Megaphone | 1 week before launch |
| Revenue setup | Cashflow | Before first customer |
| New agent | Warden | Per pipeline |

---

## ESCALATION TO STACKZ

Escalate when:
- 2+ weeks behind milestones
- Critical dependency blocked
- Metrics show non-viability
- Pivot needed (scope change)
- Ready to graduate
