# VENTURE INTAKE
*How a new venture gets set up in Startup Team*

---

## TRIGGER

A new venture starts when Stackz approves a Radar proposal and routes it to Founder.

---

## INTAKE CHECKLIST

### Day 1: Receipt

- [ ] Receive approved proposal from Stackz
- [ ] Confirm venture name and slug
- [ ] Confirm budget allocation
- [ ] Confirm timeline expectations

### Day 2: Setup

- [ ] Create `startup/ventures/[slug]/` folder
- [ ] Create VENTURE_BRIEF.md from proposal
- [ ] Set up metrics tracking (or confirm existing)
- [ ] Create task/project board

### Day 3-5: Team Assembly

- [ ] Request agents from Warden if needed
- [ ] Brief Forge on technical requirements
- [ ] Brief Canvas on brand requirements
- [ ] Set up communication channels

### Day 6-7: Kickoff

- [ ] Discovery sprint kickoff
- [ ] Initial team sync
- [ ] First sprint planning
- [ ] Confirm MVP scope

---

## DOCUMENTATION

### VENTURE_BRIEF.md Contents
- Venture name and description
- Target customer
- Value proposition
- Revenue model
- MVP scope
- Team assignments
- Timeline
- Budget

---

## INITIAL METRICS TO TRACK

| Metric | Target | How Measured |
|--------|--------|--------------|
| MRR | $X | Stripe/Payment |
| Customers | X | Customer count |
| Churn | < 5% | % leaving monthly |
| Signups | X | New registrations |
| Activation | > 40% | Reach "aha" moment |

---

## ESCALATION

If intake blocked:
- Missing proposal details → Stackz
- Team availability → Warden
- Budget questions → Cashflow
