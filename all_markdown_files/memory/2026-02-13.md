## Architecture Decisions

### Clearance Tier System
```
Tier 0 (none)      → No external access
Tier 1 (read_only) → Read APIs/web, no writes
Tier 2 (read_write)→ Post/publish capabilities
Tier 3 (financial) → View financial data (future)
Tier 4 (admin)     → Full access (Stackz, Warden, credentials-mgr only)
```

### Bootstrap Approach
- **Phase 1:** HR Foundation (warden, credentials-mgr, auditor, recruiter)
- **Phase 2:** Team Leads (5 agents across all departments)
- **Phase 3:** Team Members (as needed, requested by team leads)

**Rationale:** Establish governance and security before scaling

### Lane Queue Protocol
All inter-team communication routed through Stackz's lane queue:
```
Team A Agent → Team A Lead → Stackz → Team B Lead → Team B Agent
```
**No direct cross-team contact** — ensures oversight and coordination
